# A2 任务教学：AUX ADC → DAC（含降噪调优，零基础小白版）

> **适用对象**：第一次接触嵌入式音频开发的同学，已经完成了 [A1 任务教学](BT8920A2音频实验教学.md)。
> **目标**：理解"AUX 模拟输入 → ADC → DMA → 中断 → DAC → 耳机输出"整条链路，并掌握如何调试噪声。
> **配套工程**：[`app/`](../../app/)；详细文档参考 [`SDK入门与任务实施指南.md`](SDK入门与任务实施指南.md)。

## 第 0 节 · A2 任务在做什么？

**一句话**：把 AUX 输入的模拟信号（如电脑音频），通过芯片内部 ADC 采样，再从 DAC 输出到耳机，**让耳机播放电脑的声音**。

```text
电脑音频 (1 kHz 正弦波)
  ↓ AUX 接口 (PE6/PE7)
SDADC 模拟通路 + 数字采样
  ↓ DMA 搬运 (512 sample = 32 ms)
buf_auxadc[512]
  ↓ DMA 半中断 / 全中断
auxadc_isr()
  ↓ __wrap_auxadc_pcm_to_dac (本任务重写)
DAC FIFO
  ↓ DAC 模拟输出
耳机
```

## 第 1 节 · 关键技术概念

### 1.1 ADC（模数转换器）

ADC = 把连续的模拟电压变成离散的数字代码。

```text
模拟信号:        ~~~~ (连续曲线)
                  ↓
ADC 采样:        | | | | | | | (每秒 N 个数字)
                  ↓
数字代码:        1024, 2048, 1024, -1024, ...
```

SDADC 是 **Sigma-Delta ADC**，对音频信号质量更好，集成在 BT8920A2 内部。

### 1.2 DMA（直接内存访问）

**问题**：CPU 每搬一个样点就做一次事，负担很重。

**解决**：让 DMA 硬件自动从 ADC 搬到内存，CPU 干别的。

**机制**：
- 设好起始地址 `SDADCLDMAADDR`、样点数 `SDADCLDMASIZE`；
- DMA 自动搬，到一半触发 **Half Done 中断**，搬完触发 **Done 中断**；
- ISR 一次性处理一批样点（不是每个样点都中断）。

### 1.3 缓冲区布局

```c
u32 buf_auxadc[512];   // 512 个 u32 = 2048 字节
```

**双声道 16-bit 立体声排列**：

```text
offset:  0    2    4    6    8   ...  1022  1024  1026
value:  L0   R0   L1   R1   L2  ...  L255  R255  ???
```

- 每个 sample 占 2 字节（16-bit）；
- 立体声对 L+R 占 4 字节；
- 512 个 u32 = 128 对 L+R = **128 个立体声 sample**；
- 但 `samples = 512` 表示 **每个通道 512 个 sample**（双声道合计 1024 个 sample = 2048 字节），但 `buf_auxadc[512]` 只有 2048 字节——**刚好够**！

### 1.4 中断回调机制

```c
void auxadc_isr(void) {           // 每 ~32 ms 触发一次
    if (SDADCDMAFLAG & BIT(1)) {   // Half Done
        SDADCDMACLR = BIT(1);      // 清中断标志
        auxadc_pcm_to_dac(1, buf, samples);  // 处理第一半
    } else if (SDADCDMAFLAG & BIT(0)) {  // Done
        SDADCDMACLR = BIT(0);      // 清中断标志
        auxadc_pcm_to_dac(0, buf, samples);  // 处理第二半
    }
}
```

ISR 调用 `auxadc_pcm_to_dac()` 把 ADC 数据搬到 DAC FIFO。

### 1.5 弱符号 vs 强符号

- **弱符号 (weak)**：可以被同名 strong 符号覆盖；
- **强符号 (strong)**：直接覆盖会冲突。

`auxadc_pcm_to_dac` 在库中是 **strong**——我们要覆盖它需要特殊方法。

### 1.6 GCC `--wrap` 选项

GCC 链接器选项：把库代码里所有 `xxx` 的引用替换为 `__wrap_xxx`。

```c
// 库代码里调用: auxadc_pcm_to_dac(...)
// 链接器重写: -> __wrap_auxadc_pcm_to_dac(...)
// 库里的 auxadc_pcm_to_dac(...) 实现被忽略
```

**使用方法**：

1. 应用层实现 `__wrap_auxadc_pcm_to_dac`：
   ```c
   void __wrap_auxadc_pcm_to_dac(u8 flag, u8 *adc_buf, u16 adc_samples) { ... }
   ```
2. 链接器加选项：
   ```
   --wrap=auxadc_pcm_to_dac
   ```

## 第 2 节 · A2 任务三大难点（顺序踩坑）

### 2.1 难点 1：库版本是强符号

**报错**：
```
audio.c:(.com_text.isr.sdadc+0x0): multiple definition of `auxadc_pcm_to_dac'
bsp_adc_pcm_to_dac_ext.c:(.com_text.isr.sdadc+0x0): first defined here
```

**解决**：用 GCC `--wrap` 链接器选项。

**Code::Blocks 配置**：[app.cbp:40](../../app/projects/standard/app.cbp)

```xml
<Add option="--wrap=auxadc_pcm_to_dac" />
```

> **注意**：链接器是 ld 直接调用（不是 gcc），所以必须写 `--wrap=`，**不要**写 `-Wl,--wrap=`（CB 会原样传给 ld，ld 不认识 `-Wl,` 前缀）。

### 2.2 难点 2：AUX 引脚配置错误

**症状**：配置 `auxadc_cb.channel = 0x33`（PE6/PE7）后，串口打印仍是 `channel = 0x11`（PA6/PA7）——配置没生效。

**原因**：`bsp_adc_aux_ext.c` 里 `auxadc_analog_channel_select_ext()` 函数**默认只支持 PA6/PA7**，其他分支被注释掉。即使配置 channel=PE6/PE7，模拟通路也没切换。

**修复**：放开所有分支：

```c
// bsp_adc_aux_ext.c:181
if (left) {
    if (left == CH_AUXL_PA6) {
        AUANGCON7 |= BIT(8);                    // PA6
    }
    else if (left == CH_AUXL_PB1) {
        AUANGCON7 |= BIT(9);                    // PB1
    }
    else if (left == CH_AUXL_PE6) {
        AUANGCON7 |= BIT(10);                   // PE6 (AUXL2) ← 关键
    }
    else if (left == CH_AUXL_PF4) {
        AUANGCON7 |= BIT(11);                   // PF4
    }
    else if (left == CH_AUXL_VCMBUF) {
        AUANGCON7 |= BIT(19);                   // VCMBUF
    }
    else if (left == CH_AUXL_VOUTRP) {
        AUANGCON7 |= BIT(15);                   // VOUTRP
    }
}
```

右声道同理（PE7 → `AUANGCON8 |= BIT(10)`）。

### 2.3 难点 3：AUX 引脚**不是 PA6/PA7**

**问题**：默认代码用 `CH_AUXL_PA6 | CH_AUXR_PA7`（0x11），但原理图显示 BT8920A2 的 AUX 输入实际是 **PE6/PE7**。

**引脚定义图确认**：

| 声道 | 实际引脚 | AUX 名 | ADC 通道 |
|---|---|---|---|
| 左 | **PE6** | **AUXL2** | **ADC8** |
| 右 | **PE7** | **AUXR2** | **ADC9** |

**修复**：

```c
// bsp_adc_aux_ext.c:121
auxadc_cb.channel = CH_AUXL_PE6 | CH_AUXR_PE7;  // 0x03 | 0x30 = 0x33
```

### 2.4 难点 4：DAC FIFO 持续 575 死锁

**症状**：`DAC FIFOCNT = 575/575` 恒定不变，FIFO 永远满 → ADC ISR 推不进去 → DAC 不消费 → 死锁。

**根因**：`dac_obuf_init()` 里写了 `AUBUFCON |= BIT(0)`（Reset audio Buffer），**Reset 状态需要写 0 退出**。A1 测试中因为有库函数会清零，A2 模式没调用那个函数，FIFO 一直 reset。

**修复**：

```c
// bsp_dac_ext.c:46
void dac_obuf_init(void)
{
    ...
    AUBUFCON |= BIT(0);   // 进入 reset
    ...
    AUBUFCON &= ~BIT(0);  // 退出 reset (关键!)
}
```

## 第 3 节 · A2 降噪调优：5 个调试点

### 3.1 调优过程时间线

| 步骤 | 改动 | 结果 |
|---|---|---|
| PC7 | 基础打通（PE6/PE7 + FIFO reset 修复）| **大噪声 + 频率不稳** |
| PC8 | 降低 ADC 增益 + 关闭 ISR print | 噪声减小，**声音变小** |
| PC9 | `avol=40`（调大补偿） | 仍有噪声 |
| PC10 | `dac_spr_set(SPR_16000)`（DAC 走 16 kHz + SRC）| 频率**严重不稳**（100~300 Hz 跳变）|
| PC11 | `auxadc_cb.sample_rate = SPR_16000`（ADC 与 DAC 同速）| 仍有噪声 |
| **PC12** | `samples=512` + `avol=53` + ADC/DAC 同 16 kHz | **1 kHz 稳定 + 噪声小 + 声音大** ✅ |

### 3.2 调优点 1：降低 SDADC 增益

**问题**：原 `gain = (15 << 6) | 30`（模拟 15 + 数字 30）太接近满量程，**易削顶**。

**修复**：

```c
// bsp_adc_aux_ext.c:124
auxadc_cb.gain = (8 << 6) | (15);   // 模拟 8 (-3 dB) + 数字 15 (-15 dB)
```

**为什么**：
- 模拟增益范围 0~23（每档约 3 dB）；
- 数字增益范围 0~31（每档约 1 dB）；
- **总增益降低约 20 dB**，信号幅度变小约 10 倍；
- **但远离削顶**，量化噪声相对变小。

### 3.3 调优点 2：调大 DAC 模拟音量

**问题**：调低 SDADC 增益后声音变小。

**修复**：

```c
// bsp_adc_aux_ext.c:327
dac_set_avol(53);  // N_1DB = -1 dB，接近 0 dB 临界
```

**avol 对照表**：

| avol 索引 | 编码 | dB | 描述 |
|---:|---|---|---|
| 30 | `N_24DB` | -24 | PC8 太小 |
| 40 | `N_14DB` | -14 | PC9 中间 |
| **50** | **`N_4DB`** | **-4** | **PC7** |
| **53** | **`N_1DB`** | **-1** | **PC12（接近上限）** |
| 54 | `N_0DB` | 0 | 临界，可能削顶 |
| 55 | `P_1DB` | +1 | **削顶警告** |
| 59 | `P_5DB` | +5 | **必削顶** |

### 3.4 调优点 3：ADC 与 DAC 同速

**核心教训**：**SRC 模块会引入延迟，不要用 SRC**！

```c
// bsp_adc_aux_ext.c:122 (PC11 修复)
auxadc_cb.sample_rate = SPR_16000;  // ADC 16 kHz
dac_spr_set(SPR_16000);             // DAC 16 kHz  (避免 SRC)
```

**为什么**：
- DAC 输出时钟是芯片内部的，DAC 输入时钟是 ADC 推入的；
- 两边速率不同时，**DAC 内部 SRC 模块要做采样率转换**；
- SRC 转换引入**额外延迟和不确定性**；
- 长时间运行后，FIFO 缓冲会**累积延迟差** → 频率漂移。

**正确做法**：让 ADC 和 DAC **用同一个采样率**，跳过 SRC。

### 3.5 调优点 4：增大 samples

**问题**：samples=256 时 ADC ISR 每 ~5.8 ms 触发一次（172 Hz），FIFO 频繁被填满 → 容易过载。

**修复**：

```c
// bsp_adc_aux_ext.c:123
auxadc_cb.samples = 512;   // 半 DMA 中断间隔 ~32 ms（31 Hz）
```

**缓冲区大小确认**：

```text
总数据量 = samples × 2 声道 × 2 字节 = 512 × 4 = 2048 字节
buf_auxadc[512] = 512 × 4 = 2048 字节  ← 刚好够
```

**注意**：buf 必须够大！否则 DMA 会越界写入未定义内存。

### 3.6 调优点 5：FIFO 余量

**PC7 时**：FIFO 持续 574/575（满），DAC 不消费 → 数据被丢弃。
**PC12 后**：FIFO 应该降到 200~500 之间波动（具体取决于测量的瞬间）。

## 第 4 节 · A2 任务最终代码

### 4.1 应用层 __wrap_auxadc_pcm_to_dac

[app/bsp_ext/bsp_adc_pcm_to_dac_ext.c](../../app/bsp_ext/bsp_adc_pcm_to_dac_ext.c)：

```c
#include "include.h"
#include "bsp_dac_ext.h"

// 使用 GCC --wrap 链接器选项：库代码里所有 auxadc_pcm_to_dac 调用
// 会自动改写到 __wrap_auxadc_pcm_to_dac（本函数）。
//
// DAC FIFO 状态:
//   AUBUFCON BIT(8) = 1 表示 FIFO 已满
//   AUBUFCON BIT(8) = 0 表示未满可写
//   AUBUFCON BIT(0) = 1 表示 FIFO reset（冻结）
//
//SDADC DMA 双缓冲布局（双声道，512 samples per half）：
//   buf[0..1023] = L0 R0 L1 R1 ... L511 R511 (16-bit 双声道交错)

AT(.com_text.isr.sdadc)
void __wrap_auxadc_pcm_to_dac(u8 flag, u8 *adc_buf, u16 adc_samples)
{
    u16 *buf16 = (u16 *)adc_buf;
    u16 samples = adc_samples;

    // 关键：退出 DAC FIFO reset 状态，否则 FIFO 冻结不会消费
    AUBUFCON &= ~BIT(0);

    for (u16 i = 0; i < samples; i++) {
        if (AUBUFCON & BIT(8)) {
            // FIFO 已满，跳过剩余（会发生断续但不会卡死）
            break;
        }
        u16 left  = buf16[i * 2 + 0];
        u16 right = buf16[i * 2 + 1];
        AUBUFDATA = ((u32)right << 16) | (u32)left;
    }
}
```

{

## 第 5 节 · A2 测试方法

### 5.1 物理接线

```text
电脑音频输出口（TRS 3 段插头）
  ↓
  Tip（左声道） → 开发板 PE6
  Ring（右声道）→ 开发板 PE7
  Sleeve（地）  → 开发板 GND

开发板耳机座 → 耳机
```

**重要**：
- AUX 输入和耳机输出**不能接同一个口**；
- 信号源地和开发板地必须共地；
- 信号源音量调 30-50%，避免削顶。

### 5.2 测试步骤

1. **Build** Code::Blocks，生成新 `app.dcf`；
2. **烧录** 到开发板；
3. **电脑播放 1 kHz 正弦波**（用 Audacity 生成 WAV）；
4. **示波器**：
   - CH1 接 DAC 输出脚；
   - 时基 `500 µs/div`；
   - `Auto Scale` 后 `Meas` 加 `Freq`；
   - **期望读数 ~1 kHz，频率稳定不跳**；
5. **耳机**听声音是否清晰。

### 5.3 串口关键观察

| 字段 | 期望 | 含义 |
|---|---|---|
| `auxadc_cb.channel` | `0x33` | PE6/PE7 配通 |
| `DAC FIFOCNT` | 200~500 之间波动 | FIFO 没死锁 |
| `DACDIGCON0` | `0x219` | DAC 走 16 kHz 直通 |
| `SampleRate` | ~16000 | ADC 16 kHz，与 DAC 一致 |

### 5.4 通过标准

- ✅ 示波器 Freq ≈ 1 kHz，**稳定不跳**；
- ✅ 串口 `DAC FIFOCNT` 在波动范围（不是持续 575）；
- ✅ 耳机能听到 AUX 输入的 1 kHz 音调；
- ✅ 声音清晰（噪声小）。

## 第 6 节 · 5 个常见故障排查

### 6.1 `multiple definition of auxadc_pcm_to_dac`

**解决**：加 `--wrap=auxadc_pcm_to_dac` 链接器选项，应用层实现 `__wrap_auxadc_pcm_to_dac`。

### 6.2 `unrecognized option '-Wl,--wrap=...'`

**原因**：链接器直接是 ld（不是 gcc），不认识 `-Wl,` 前缀。

**解决**：写 `--wrap=...`（不要 `-Wl,`）。

### 6.3 串口 `auxadc_cb.channel = 0x11`（不是 0x33）

**原因 1**：`main.c` 没强制 `xcfg_cb.test_mode = TEST_AUX_ADC2DAC`，xcfg 配置可能不是 1。

**原因 2**：`aux_analog_channel_select_ext()` PE6/PE7 路径被注释。

**解决**：放开 if 分支 + 确认 channel 值。

### 6.4 `DAC FIFOCNT` 持续 575 不动

**原因**：`AUBUFCON BIT(0)` reset 状态没退出。

**解决**：`dac_obuf_init()` 末尾加 `AUBUFCON &= ~BIT(0)`。

### 6.5 频率不稳（100~300 Hz 跳变）

**原因**：SRC 模块引入延迟。

**解决**：让 ADC 与 DAC **同采样率**（都用 SPR_16000）。

## 第 7 节 · 关键概念对照

| 概念 | 一句话解释 |
|---|---|
| ADC | 把模拟电压变成数字代码 |
| DMA | 硬件自动搬数据，不占 CPU |
| 双缓冲 | DMA 在两段缓冲区循环搬，每段触发一次中断 |
| FIFO | DAC 内部小缓冲，存待播放样点 |
| ISR | 中断服务程序，硬件事件触发 |
| 弱符号 | 可被覆盖的全局符号 |
| 强符号 | 不可被覆盖，会冲突 |
| `--wrap` | GCC 链接器选项，重写符号引用 |
| SRC | DAC 内部采样率转换模块 |
| avol | DAC 模拟音量（0~59） |
| dvol | DAC 数字音量 |
| samples | ADC DMA 半中断的样点数 |
| gain | ADC 增益（模拟+数字） |
